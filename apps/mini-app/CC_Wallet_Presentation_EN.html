Security

Security Architecture

Security Philosophy: Defense in depth with client-side cryptography. Private keys are generated, encrypted, and stored only on the user's device. Only signed transactions are transmitted to the network.

1

Client-Side Only Cryptography

Private keys never leave the device. All signing operations happen locally using WebCrypto API.

2

Reproducible Builds

Open-source codebase with deterministic builds. Anyone can verify deployed code matches source.

3

Formal Verification

Critical cryptographic functions are mathematically proven correct using formal verification methods.

4

Open Source + Community Audit

Full codebase transparency enables continuous community review and third-party security audits.

5

Canton Protocol Security

Inherits Canton's formal verification (Isabelle/HOL), BFT consensus, and end-to-end encryption.

Key Protection

  • AES-256-GCM at-rest encryption
  • PIN-derived key encryption
  • Biometric unlock support
  • Secure enclave when available

Transaction Security

  • Transaction signing confirmation
  • Amount and recipient verification
  • Rate limiting on sends
  • Replay attack protection

Innovation

Seedless Wallet Architecture

Problem: Traditional seed phrases are the #1 cause of crypto asset loss. Users either lose them, expose them, or never properly back them up. This creates a fundamental barrier to mainstream adoption.

Our Solution: PIN + Biometric + Cloud Recovery

Key Generation

  • ECDSA P-256 key pair generated client-side
  • Private key encrypted with PIN-derived key
  • PBKDF2 with 100,000 iterations
  • Encrypted blob stored in Telegram CloudStorage

Recovery Mechanism

  • Same Telegram account + PIN = Full recovery
  • Works on any device
  • No seed phrase to lose or expose
  • KYC Level 2 enables PIN reset

Security Comparison

Feature Traditional Seed CC Bot Seedless
Backup Method Paper/Metal/Memory Encrypted Cloud + PIN
Recovery Complexity 12-24 words required Just 6-digit PIN
Phishing Risk High (seed exposure) Low (PIN alone useless)
Cross-Device Manual seed entry Automatic via Telegram
Self-Custody Yes Yes (client-side keys)
